# Keep — A Google Keep Clone

A pixel-conscious clone of Google Keep's note-taking interface, built with **plain HTML, CSS, and vanilla JavaScript** — no frameworks, no build tools, no dependencies.

---

## Running the project

There is no installation, build step, or server required.

1. Download or clone the project folder.
2. Open `index.html` directly in any modern browser (double-click it, or drag it into a browser window).

That's it — the app runs entirely client-side.

### Optional: run it through a local server

Opening the file directly works fine for this project since it has no external API calls, but if you prefer serving it over `http://` instead of `file://` (useful if you later add features that require it), any static server works:

```bash
# Python 3
python3 -m http.server 8000

# Node (if you have npx available)
npx serve .
```

Then visit `http://localhost:8000` in your browser.

## Project structure

```
Project/
│── index.html          # Semantic markup: header, sidebar, notes/archive sections, modal, dialogs
│── css/
│     styles.css        # All styling — variables, reset, layout, components, utilities, media queries
│── js/
│     script.js         # All app logic — state, storage, rendering, CRUD, event wiring
│── assets/
│     icons/            # Reserved for any future static icon assets (icons are currently inline SVG)
│     images/           # Reserved for any future static image assets
```

No icon or image files are actually required at runtime — every icon in the UI (search, archive, pin, palette, etc.) is drawn as inline SVG directly in `index.html` / generated by `script.js`, so the app has zero external asset dependencies.

## Features

- **Create notes** via the "Take a note…" bar or the note editor modal, with a title and content.
- **Edit notes** by clicking any note card.
- **Pin / unpin notes** — pinned notes are grouped into a "Pinned" section above "Others."
- **Archive / unarchive notes** — archived notes move to their own Archive section and disappear from the active list.
- **Delete notes** with a confirmation dialog before the action is final.
- **Duplicate notes** ("Make a copy") from the note's kebab (⋮) menu.
- **Color-code notes** using the background color palette, shared between the modal and each card.
- **Search** notes live by title or content, with a "no matching notes" empty state.
- **List / grid view toggle** for the notes layout.
- **Empty states** for no notes and no search results.
- **Responsive layout** — sidebar collapses into a bottom tab bar on mobile, notes reflow into a single column.
- **Persistent storage** — all notes are saved to `localStorage`, so your notes are still there after a refresh or browser restart.

### Decorative-only UI

To visually match the real Google Keep interface, a few chrome elements are present but intentionally **not wired up** to real functionality (they show a small "isn't available in this demo" toast when clicked): *Reminders*, *Edit labels*, *Trash*, *Collaborator*, *Add image*, *Settings*, and the *Google apps* grid. This keeps the UI faithful to Keep without overstating what a vanilla HTML/CSS/JS clone can reasonably do.

## How data is stored

Notes are kept in a single `localStorage` key (`keeper_notes_v1`) as a JSON array. Each note object looks like this:

```json
{
  "id": "n_m3x1a2_bc93kd",
  "title": "Groceries",
  "content": "Milk, eggs, bread",
  "archived": false,
  "pinned": false,
  "createdAt": 1735680000000,
  "color": "note-mint"
}
```

Because everything lives in `localStorage`, notes are scoped to the browser and device you're using — there's no account system or cloud sync (this is a front-end clone, not a backend service).

**To reset the app to a blank state**, open your browser's DevTools console on the page and run:

```js
localStorage.removeItem('keeper_notes_v1');
```

then refresh the page.

## Browser support

Built with standard, broadly-supported web APIs (CSS custom properties, `localStorage`, CSS multi-column layout, flexbox). It should work in any current version of Chrome, Firefox, Safari, or Edge. No polyfills are included or required.

---

## Notes on scope

This project is a front-end learning/demo clone. It does not persist data anywhere beyond the local browser, has no authentication, and does not replicate every Google Keep feature (reminders, labels, real-time collaboration, drawings, and image uploads are visual-only placeholders). It's meant to demonstrate clean, dependency-free HTML/CSS/JS architecture and close visual fidelity to Keep's interface — not to be a production note-taking service.
